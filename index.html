DESIGN SPEC · NORTH STAR

A design spec, written for operators.

This companion document translates the homepage into a reusable product language — what every screen inherits, how components relate, and the order to ship changes in.

◆ NORTH STAR

Every surface should answer one question: "Is this vulnerable, and can I profitably take it?" — before the operator commits attention, capital, or inventory.

POSITIONING

Offensive, not observational.

Language, color, and motion should feel like a firing solution — not a dashboard. Every screen has a verdict, a target, and a next move.

SURFACE LOGIC

Injected > hosted.

Intelligence appears where operators already work (Amazon, then Cockpit). Dashboards are for managing state, never for first-touch discovery.

MATERIAL

Glass, not flat.

Deep-space blue base, cyan intelligence glow, neon-green verdicts, red for defended states. Layered glass panels give hierarchy without decoration.

SCREEN-BY-SCREEN

Every surface, spec'd.

Five core screens, each with purpose, user mindset, hierarchy, layout, components, UX improvements, visual tone, and responsive notes.

IMPLEMENTATION ROADMAP

Three phases. Ship high-leverage work first.

Ordered by impact density — what changes the way the product reads fastest, for the lowest implementation cost.

PHASE 01

Fast wins

Week 1 · ~3 engineer-days
  • 01Adopt the token palette — bg #0a0e1a, accent #38bdf8, success #4ade80, danger #f87171. Replace all raw hex usage across the app.UI
  • 02Ship the Apex Pill component. Replace every existing score readout in one PR.UI
  • 03Force DM Sans + Instrument Serif via one global stylesheet. Remove all other font references.UI
  • 04Replace all "View / Monitor / Track" microcopy with operator verbs (Analyze / Open / Run / Execute).UX
  • 05Set `color-scheme: dark` globally; remove any lingering light-mode branches.UI
  • 06Ship the Margin Readout component everywhere dollars currently appear.UI
PHASE 02

Structural cleanup

Weeks 2–4 · ~2 engineer-weeks
  • 07Extract shared glass-panel primitive. Audit every card-like surface in the app and migrate — one per sprint-day.ARCH
  • 08Collapse all bespoke tables into one DenseTable component with sorted/virtualized rows.ARCH
  • 09Unify Breakdown into one component rendered in three modes: popover (Amazon), drawer (Feed), inline (Cockpit).ARCH
  • 10Replace legacy chart pages. Time-series charts are acceptable only inside Breakdown, never as a primary surface.UX
  • 11Introduce motion tokens (120 / 240 / 800ms) as CSS custom properties. Delete all one-off durations.UI
  • 12Responsive audit: every screen must have a tested mobile layout with bottom-sheet pattern for Breakdown.UX
PHASE 03

Premium polish

Weeks 5–8 · ~4 engineer-weeks
  • 13Ambient scanline grid + mouse-reactive glow on marketing surfaces only (never in-product).UI
  • 14Keyboard nav everywhere: j/k over Target Feed rows, Cmd+Enter executes in Cockpit, Esc always closes.UX
  • 15Saved scenarios in Cockpit with named variants and diff banner ("what changed since I last opened this").UX
  • 16Real-time Target Feed sync via websockets; optimistic updates with scanline pulse on stale rows.ARCH
  • 17Defense Matrix hover-to-explain with live PPC CPC inputs bound to sourcing profile.UX
  • 18Browser-extension polish: graceful DOM resilience, highest-Apex keyboard jump, telemetry for breakage.ARCH
FIRST 10 UI CHANGES

Ship these ten things this month.

Sorted by impact-to-effort ratio. Anyone on the team can pick one up and close it inside a day.

  1. 01Adopt the dark token palette globally. One CSS file. Delete every light-mode fallback. Single largest perceived quality jump.UI
  2. 02Ship Apex Pill. It's the only signal that matters. One component, one color map, three states.UI
  3. 03Replace display font with Instrument Serif. Only on H1/H2. Body stays DM Sans. No other display faces.UI
  4. 04Rewrite CTAs as operator verbs. "Get Started Free" / "Start Pro" / "Open Blueprint" / "Run Launch Math". Ban "View" and "Monitor".UX
  5. 05Kill every decorative chart on the primary dashboard. Replace with Target Feed table. Charts go inside Breakdown only.UX
  6. 06Add injected Apex Score pills to the Amazon extension. This is the product's defining moment of perceived value.UI
  7. 07Ship Margin Readout. Every dollar + percentage block becomes this component, including the final CTA card.UI
  8. 08Add Safe / Defended / Moat Weak status chips. Place one on every Target, one on every Blueprint.UI
  9. 09Remove all emoji, all rounded-illustration empty states. Replace with one-verb CTA. This alone removes 80% of the "generic SaaS" feel.UX
  10. 10Ship the new homepage. The one rendered in this project. Ship it to keen.technology root before any in-product change lands.UI
